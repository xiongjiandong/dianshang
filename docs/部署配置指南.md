# 部署配置指南

## 一、概述

本文档描述如何将PayPal支付系统部署到生产环境。

---

## 二、生产环境准备

### 2.1 服务器要求

| 项目 | 最低配置 | 推荐配置 |
|------|----------|----------|
| CPU | 1核 | 2核+ |
| 内存 | 1GB | 2GB+ |
| 存储 | 20GB | 50GB+ |
| 带宽 | 1Mbps | 5Mbps+ |

### 2.2 软件环境

| 软件 | 版本 |
|------|------|
| Node.js | 18.x LTS |
| MySQL | 8.0 |
| Nginx | 1.20+ |
| PM2 | 最新版 |

### 2.3 域名和SSL

1. 准备一个域名（如 `example.com`）
2. 申请SSL证书（推荐Let's Encrypt免费证书）
3. 配置HTTPS强制跳转

## 三、PayPal 生产配置

### 3.1 创建生产应用

1. 登录 [PayPal开发者平台](https://developer.paypal.com/)
2. 切换到 **Live** 标签
3. 创建新应用或使用现有应用
4. 获取 **生产Client ID** 和 **Client Secret**

### 3.2 配置生产Webhook

1. 在应用详情页点击 **Add Webhook**
2. 输入Webhook URL：
   ```
   https://yourdomain.com/api/payments/webhook
   ```
3. 选择订阅事件：
   - `PAYMENT.CAPTURE.COMPLETED`
   - `PAYMENT.CAPTURE.DENIED`
   - `PAYMENT.CAPTURE.REFUNDED`
4. 保存后记录 **Webhook ID**

### 3.3 生产凭据清单

| 凭据 | 用途 | 存储位置 |
|------|------|----------|
| Client ID | 前端SDK初始化 | 前端环境变量 |
| Client ID | 后端API调用 | 后端环境变量 |
| Client Secret | 后端API认证 | 后端环境变量 |
| Webhook ID | 验证Webhook签名 | 后端环境变量 |

## 四、后端部署

### 4.1 目录结构

```
/opt/dianshang/
├── backend/
│   ├── src/
│   ├── node_modules/
│   ├── package.json
│   ├── .env.production
│   └── ecosystem.config.js
└── logs/
    ├── app.log
    └── error.log
```

### 4.2 环境变量配置

创建 `/opt/dianshang/backend/.env.production`：

```bash
# 应用配置
NODE_ENV=production
PORT=3000
API_BASE_URL=https://yourdomain.com

# 数据库配置
DB_HOST=localhost
DB_PORT=3306
DB_NAME=dianshang
DB_USER=dianshang_user
DB_PASSWORD=your_secure_password

# PayPal生产配置
PAYPAL_PRODUCTION_CLIENT_ID=your_production_client_id
PAYPAL_PRODUCTION_CLIENT_SECRET=your_production_client_secret
PAYPAL_WEBHOOK_ID=your_webhook_id

# 安全配置
JWT_SECRET=your_secure_jwt_secret_at_least_32_characters
CORS_ORIGIN=https://yourdomain.com

# 前端配置
FRONTEND_URL=https://yourdomain.com
SITE_NAME=My Store
```

### 4.3 PM2 配置

创建 `ecosystem.config.js`：

```javascript
module.exports = {
  apps: [{
    name: 'dianshang-api',
    script: './src/app.js',
    instances: 'max',
    exec_mode: 'cluster',
    env_production: {
      NODE_ENV: 'production',
      PORT: 3000
    },
    error_file: '/opt/dianshang/logs/error.log',
    out_file: '/opt/dianshang/logs/app.log',
    log_date_format: 'YYYY-MM-DD HH:mm:ss',
    merge_logs: true,
    max_memory_restart: '500M',
    autorestart: true,
    watch: false
  }]
};
```

### 4.4 部署步骤

```bash
# 1. 安装依赖
cd /opt/dianshang/backend
npm install --production

# 2. 创建日志目录
mkdir -p /opt/dianshang/logs

# 3. 启动服务
pm2 start ecosystem.config.js --env production

# 4. 保存PM2配置
pm2 save

# 5. 设置开机启动
pm2 startup
```

### 4.5 常用命令

```bash
# 查看状态
pm2 status

# 查看日志
pm2 logs dianshang-api

# 重启服务
pm2 restart dianshang-api

# 停止服务
pm2 stop dianshang-api
```

## 五、前端部署

### 5.1 构建配置

创建 `frontend/.env.production`：

```bash
# 生产环境API
VITE_API_BASE_URL=https://yourdomain.com

# PayPal Client ID（生产）
VITE_PAYPAL_CLIENT_ID=your_production_client_id
```

### 5.2 构建命令

```bash
# 进入前端目录
cd frontend

# 安装依赖
npm install

# 构建生产版本
npm run build
```

```
├── index.html
├── assets/
│   ├── index-[hash].js

│   └── ...
└── favicon.ico
```

## 六、Nginx 配置

### 6.1 安装Nginx

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install nginx

# CentOS
sudo yum install epel-release
sudo yum install nginx
```

### 6.2 配置文件

创建 `/etc/nginx/sites-available/dianshang`：

```nginx
# HTTP重定向到HTTPS
server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;
    return 301 https://$server_name$request_uri;
}

# HTTPS配置
server {
    listen 443 ssl http2;
    server_name yourdomain.com www.yourdomain.com;

    # SSL证书
    ssl_certificate /etc/letsencrypt/live/yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/yourdomain.com/privkey.pem;
    
    # SSL配置
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256;
    ssl_prefer_server_ciphers off;
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 1d;

    # 安全头
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;

    # 前端静态文件
    root /opt/dianshang/frontend/dist;
    index index.html;

    # 前端路由（Vue Router history模式）
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API代理
    location /api {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
        proxy_read_timeout 60s;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # Gzip压缩
    gzip on;
    gzip_vary on;
    gzip_min_length 1024;
    gzip_proxied any;
    gzip_types text/plain text/css text/xml text/javascript application/javascript application/json application/xml;
    gzip_comp_level 6;

    # 访问日志
    access_log /var/log/nginx/dianshang_access.log;
    error_log /var/log/nginx/dianshang_error.log;
}
```

### 6.3 启用配置

```bash
# 创建软链接
sudo ln -s /etc/nginx/sites-available/dianshang /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重载Nginx
sudo systemctl reload nginx
```

## 七、SSL 证书配置

### 7.1 安装Certbot

```bash
# Ubuntu/Debian
sudo apt install certbot python3-certbot-nginx

# CentOS
sudo yum install certbot python3-certbot-nginx
```

### 7.2 申请证书

```bash
# 申请证书
sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com

# 测试自动续期
sudo certbot renew --dry-run
```

### 7.3 自动续期

Certbot会自动设置定时任务，证书到期前自动续期。

## 八、数据库配置

### 8.1 创建数据库和用户

```sql
-- 创建数据库
CREATE DATABASE dianshang CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 创建用户
CREATE USER 'dianshang_user'@'localhost' IDENTIFIED BY 'your_secure_password';

-- 授权
GRANT ALL PRIVILEGES ON dianshang.* TO 'dianshang_user'@'localhost';
FLUSH PRIVILEGES;
```

### 8.2 执行迁移

```bash
# 运行数据库迁移
cd /opt/dianshang/backend
npx sequelize-cli db:migrate --env production
```

### 8.3 数据库安全

```bash
# 运行安全脚本
sudo mysql_secure_installation
```

配置项：
- 设置root密码
- 移除匿名用户
- 禁止root远程登录
- 删除测试数据库

## 九、监控和日志

### 9.1 PM2监控

```bash
# 安装pm2-logrotate
pm2 install pm2-logrotate

# 设置日志保留
pm2 set pm2-logrotate:max_size 10M
pm2 set pm2-logrotate:retain 7
```

### 9.2 日志轮转

创建 `/etc/logrotate.d/dianshang`：

```
/opt/dianshang/logs/*.log {
    daily
    rotate 7
    compress
    delaycompress
    missingok
    notifempty
    create 0640 dianshang dianshang
}
```

### 9.3 监控命令

```bash
# 查看服务状态
pm2 monit

# 查看实时日志
pm2 logs

# 查看服务器资源
pm2 info dianshang-api
```

## 十一、常见问题

### 11.1 Webhook无法接收

**检查项**：
1. Webhook URL是否可公开访问
2. SSL证书是否有效
3. 服务器防火墙是否放行
4. Nginx配置是否正确代理

### 11.2 支付按钮无法加载

**检查项**：
1. Client ID是否正确
2. 网络是否能访问PayPal CDN
3. 浏览器控制台是否有错误

### 11.3 API请求超时

**检查项**：
1. 后端服务是否正常运行
2. Nginx代理超时配置
3. 数据库连接是否正常
