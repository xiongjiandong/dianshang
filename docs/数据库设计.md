# 数据库设计文档

## 一、概述

本文档描述PayPal支付系统所需的数据表设计。

### 1.1 数据库信息

| 项目 | 值 |
|------|------|
| 数据库类型 | MySQL 8.0 |
| 字符集 | utf8mb4 |
| 排序规则 | utf8mb4_unicode_ci |
| 引擎 | InnoDB |

### 1.2 数据表列表

| 表名 | 说明 |
|------|------|
| users | 用户表 |
| products | 商品表 |
| orders | 订单表 |
| order_items | 订单项表 |
| payments | 支付记录表 |
| payment_logs | 支付日志表 |

## 二、表结构设计

### 2.1 用户表 (users)

```sql
CREATE TABLE users (
    id VARCHAR(36) PRIMARY KEY COMMENT 'UUID主键',
    email VARCHAR(255) NOT NULL UNIQUE COMMENT '邮箱',
    password_hash VARCHAR(255) COMMENT '密码哈希',
    name VARCHAR(100) COMMENT '用户名',
    phone VARCHAR(20) COMMENT '手机号',
    avatar VARCHAR(500) COMMENT '头像URL',
    status TINYINT DEFAULT 1 COMMENT '状态: 0-禁用 1-启用',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    
    INDEX idx_email (email),
    INDEX idx_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

### 2.2 商品表 (products)

```sql
CREATE TABLE products (
    id VARCHAR(36) PRIMARY KEY COMMENT 'UUID主键',
    name VARCHAR(255) NOT NULL COMMENT '商品名称',
    description TEXT COMMENT '商品描述',
    price DECIMAL(10, 2) NOT NULL COMMENT '价格',
    currency VARCHAR(3) DEFAULT 'USD' COMMENT '货币代码',
    stock INT DEFAULT 0 COMMENT '库存',
    image VARCHAR(500) COMMENT '商品图片URL',
    status TINYINT DEFAULT 1 COMMENT '状态: 0-下架 1-上架',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    INDEX idx_status (status),
    INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='商品表';
```

### 2.3 订单表 (orders)

```sql
CREATE TABLE orders (
    id VARCHAR(36) PRIMARY KEY COMMENT 'UUID主键',
    order_no VARCHAR(32) NOT NULL UNIQUE COMMENT '订单编号',
    user_id VARCHAR(36) COMMENT '用户ID（游客可为空）',
    
    -- 金额信息
    subtotal DECIMAL(10, 2) NOT NULL COMMENT '商品小计',
    shipping_fee DECIMAL(10, 2) DEFAULT 0.00 COMMENT '运费',
    tax DECIMAL(10, 2) DEFAULT 0.00 COMMENT '税费',
    discount DECIMAL(10, 2) DEFAULT 0.00 COMMENT '折扣金额',
    total_amount DECIMAL(10, 2) NOT NULL COMMENT '订单总金额',
    currency VARCHAR(3) DEFAULT 'USD' COMMENT '货币代码',
    
    -- 订单状态
    -- pending: 待支付
    -- paid: 已支付
    -- processing: 处理中
    -- shipped: 已发货
    -- delivered: 已送达
    -- cancelled: 已取消
    -- refunded: 已退款
    status ENUM('pending', 'paid', 'processing', 'shipped', 'delivered', 'cancelled', 'refunded') 
        DEFAULT 'pending' COMMENT '订单状态',
    
    -- 收货地址
    recipient_name VARCHAR(100) COMMENT '收件人姓名',
    phone VARCHAR(20) COMMENT '联系电话',
    address TEXT COMMENT '详细地址',
    city VARCHAR(100) COMMENT '城市',
    state VARCHAR(100) COMMENT '省/州',
    postal_code VARCHAR(20) COMMENT '邮编',
    country VARCHAR(3) COMMENT '国家代码',
    
    -- 备注
    notes TEXT COMMENT '订单备注',
    
    -- 时间戳
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    paid_at TIMESTAMP NULL COMMENT '支付时间',
    shipped_at TIMESTAMP NULL COMMENT '发货时间',
    delivered_at TIMESTAMP NULL COMMENT '送达时间',
    cancelled_at TIMESTAMP NULL COMMENT '取消时间',
    
    -- 外键和索引
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE SET NULL,
    INDEX idx_user_id (user_id),
    INDEX idx_order_no (order_no),
    INDEX idx_status (status),
    INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='订单表';
```

### 2.4 订单项表 (order_items)

```sql
CREATE TABLE order_items (
    id VARCHAR(36) PRIMARY KEY COMMENT 'UUID主键',
    order_id VARCHAR(36) NOT NULL COMMENT '订单ID',
    product_id VARCHAR(36) COMMENT '商品ID',
    
    -- 商品信息（冗余存储，防止商品删除后丢失）
    product_name VARCHAR(255) NOT NULL COMMENT '商品名称',
    product_sku VARCHAR(100) COMMENT '商品SKU',
    product_image VARCHAR(500) COMMENT '商品图片',
    
    -- 数量和价格
    quantity INT NOT NULL COMMENT '数量',
    unit_price DECIMAL(10, 2) NOT NULL COMMENT '单价',
    total_price DECIMAL(10, 2) NOT NULL COMMENT '小计',
    
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    FOREIGN KEY (order_id) REFERENCES orders(id) ON DELETE CASCADE,
    INDEX idx_order_id (order_id),
    INDEX idx_product_id (product_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='订单项表';
```

### 2.5 支付记录表 (payments)

```sql
CREATE TABLE payments (
    id VARCHAR(36) PRIMARY KEY COMMENT 'UUID主键',
    order_id VARCHAR(36) NOT NULL COMMENT '关联订单ID',
    
    -- PayPal相关信息
    paypal_order_id VARCHAR(100) COMMENT 'PayPal订单ID',
    paypal_transaction_id VARCHAR(100) COMMENT 'PayPal交易ID',
    paypal_payer_id VARCHAR(100) COMMENT 'PayPal付款人ID',
    
    -- 付款人信息
    payer_email VARCHAR(255) COMMENT '付款人邮箱',
    payer_name VARCHAR(255) COMMENT '付款人姓名',
    payer_country VARCHAR(3) COMMENT '付款人国家',
    
    -- 支付信息
    payment_method VARCHAR(20) DEFAULT 'paypal' COMMENT '支付方式',
    amount DECIMAL(10, 2) NOT NULL COMMENT '支付金额',
    currency VARCHAR(3) DEFAULT 'USD' COMMENT '货币代码',
    
    -- 支付状态
    -- pending: 待处理
    -- authorized: 已授权
    -- captured: 已捕获
    -- completed: 已完成
    -- failed: 失败
    -- refunded: 已退款
    -- partially_refunded: 部分退款
    status ENUM('pending', 'authorized', 'captured', 'completed', 'failed', 'refunded', 'partially_refunded')
        DEFAULT 'pending' COMMENT '支付状态',
    
    -- 错误信息
    error_code VARCHAR(50) COMMENT '错误代码',
    error_message TEXT COMMENT '错误信息',
    
    -- Webhook相关
    webhook_event_id VARCHAR(100) COMMENT 'Webhook事件ID',
    webhook_received_at TIMESTAMP NULL COMMENT 'Webhook接收时间',
    
    -- 时间戳
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    captured_at TIMESTAMP NULL COMMENT '捕获时间',
    refunded_at TIMESTAMP NULL COMMENT '退款时间',
    
    -- 外键和索引
    FOREIGN KEY (order_id) REFERENCES orders(id) ON DELETE CASCADE,
    UNIQUE INDEX idx_paypal_order_id (paypal_order_id),
    INDEX idx_paypal_transaction_id (paypal_transaction_id),
    INDEX idx_order_id (order_id),
    INDEX idx_status (status),
    INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='支付记录表';
```

### 2.6 支付日志表 (payment_logs)

```sql
CREATE TABLE payment_logs (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    payment_id VARCHAR(36) COMMENT '支付记录ID',
    
    -- 操作信息
    action VARCHAR(50) NOT NULL COMMENT '操作类型',
    request_data JSON COMMENT '请求数据',
    response_data JSON COMMENT '响应数据',
    
    -- 请求信息
    ip_address VARCHAR(45) COMMENT 'IP地址',
    user_agent VARCHAR(500) COMMENT 'User-Agent',
    
    -- 执行信息
    execution_time INT COMMENT '执行时间(ms)',
    success TINYINT DEFAULT 1 COMMENT '是否成功',
    error_message TEXT COMMENT '错误信息',
    
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    INDEX idx_payment_id (payment_id),
    INDEX idx_action (action),
    INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='支付日志表';
```

### 3.3 模型关联

```javascript
// backend/src/models/index.js
const { Sequelize } = require('sequelize');
const config = require('../config/database');

const sequelize = new Sequelize(
  config.database,
  config.username,
  config.password,
  config
);

const User = require('./User');
const Product = require('./Product');
const Order = require('./Order');
const OrderItem = require('./OrderItem');
const Payment = require('./Payment');
const PaymentLog = require('./PaymentLog');

// 用户 - 订单 (一对多)
User.hasMany(Order, { foreignKey: 'userId' });
Order.belongsTo(User, { foreignKey: 'userId' });

// 订单 - 订单项 (一对多)
Order.hasMany(OrderItem, { foreignKey: 'orderId', as: 'items' });
OrderItem.belongsTo(Order, { foreignKey: 'orderId' });

// 订单 - 支付记录 (一对多)
Order.hasMany(Payment, { foreignKey: 'orderId' });
Payment.belongsTo(Order, { foreignKey: 'orderId' });

// 支付记录 - 支付日志 (一对多)
Payment.hasMany(PaymentLog, { foreignKey: 'paymentId' });
PaymentLog.belongsTo(Payment, { foreignKey: 'paymentId' });

module.exports = {
  sequelize,
  User,
  Product,
  Order,
  OrderItem,
  Payment,
  PaymentLog
};
```

## 四、索引设计

### 4.1 主键索引

所有表使用UUID作为主键，自动创建主键索引。

### 4.2 业务索引

| 表名 | 索引名 | 字段 | 类型 | 说明 |
|------|--------|------|------|------|
| orders | idx_user_id | user_id | 普通 | 用户查询订单 |
| orders | idx_order_no | order_no | 唯一 | 订单号查询 |
| orders | idx_status | status | 普通 | 按状态筛选 |
| payments | idx_paypal_order_id | paypal_order_id | 唯一 | PayPal订单关联 |
| payments | idx_order_id | order_id | 普通 | 订单关联查询 |

## 六、数据备份

### 6.1 备份策略

- 每日全量备份
- 保留最近7天的备份
- 重要操作前手动备份

### 6.2 备份命令

```bash
# 全量备份
mysqldump -u root -p dianshang > backup_$(date +%Y%m%d).sql

# 恢复备份
mysql -u root -p dianshang < backup_20240420.sql
```
